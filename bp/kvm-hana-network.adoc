Esta seção fornece informações sobre como configurar o SR-IOV no SUSE KVM para SAP HANA usando armazenamento NetApp .



== Configuração do SR-IOV

Esta seção mostra como configurar o SR-IOV usando placas de rede Mellanox Connect-X.  É baseado em https://enterprise-support.nvidia.com/s/article/HowTo-Configure-SR-IOV-for-ConnectX-4-ConnectX-5-ConnectX-6-with-KVM-Ethernet["Portal de Suporte Empresarial da NVIDIA | Como configurar SR-IOV para ConnectX-4/ConnectX-5/ConnectX-6 com KVM (Ethernet)"] .  O guia SUSE SAP HANA KVM descreve isso com base em uma placa de rede Intel.  Recomenda-se o uso de conexões Ethernet redundantes, combinando duas portas físicas como trunk/bond.  As portas virtuais (VF) atribuídas à VM também precisam ser configuradas como trunk dentro da VM.

image:kvm-hana-image1.png["largura=493, altura=409"]

Pré-requisitos:

* O KVM está instalado.
* SR-IOV está habilitado no BIOS do servidor.
* A passagem de PCI é habilitada adicionando “intel_iommu=on” e “iommu=pt” como opções no carregador de inicialização.
* Os drivers MLNX_OFED mais recentes estão instalados nos hosts KVM e na máquina virtual.




== Habilitar SR-IOV e definir o número de VFs.

Neste capítulo, a funcionalidade SR-IOV é habilitada, a quantidade máxima de VFs (interfaces virtuais) por porta física é definida no firmware do adaptador e a quantidade de VFs utilizadas por porta física (100GbE) é definida, quatro neste exemplo.


NOTE: Cada VF atribuída a uma VM requer uma largura de banda mínima de 10 Gbit/s.  Por exemplo, não crie e atribua mais de uma VF (Virtual Function) para uma porta física de 25GbE.

. Execute o MFT (Mellanox Firmware Tools):


....
# mst start
Starting MST (Mellanox Software Tools) driver set
Loading MST PCI module – Success
Loading MST PCI configuration module – Success
Create devices
Unloading MST PCI module (unused) – Success
....
. Localize o dispositivo:


....
# mst status
MST modules:
------------
MST PCI module is not loaded
MST PCI configuration module loaded

MST devices:

------------

/dev/mst/mt4125_pciconf0 - PCI configuration cycles access.
domain:bus:dev.fn=0000:ab:00.0 addr.reg=88 data.reg=92 cr_bar.gw_offset=-1

Chip revision is: 00
....
. Verifique o estado do dispositivo:


....
mlxconfig -d /dev/mst/mt4125_pciconf0 q |grep -e SRIOV_EN -e NUM_OF_VFS
NUM_OF_VFS 8
SRIOV_EN True(1)_
....
. Se necessário, habilite o SR-IOV:


....
mlxconfig -d /dev/mst/mt4125_pciconf0 set SRIOV_EN=1
....
. Defina a quantidade máxima de VFs:


....
mlxconfig -d /dev/mst/mt4125_pciconf0 set  NUM_OF_VFS=4
....
. Reinicie o servidor se o recurso precisar ser ativado ou se a quantidade máxima de VFs tiver sido alterada.

=== Habilitando VFs durante a inicialização

A configuração acima não persiste após a reinicialização.  Para tornar a alteração persistente, é necessário criar um arquivo de unidade _systemd_, localizado em _/etc/systemd/system/after.local_, com o seguinte conteúdo:

....
[Unit]
Description=/etc/init.d/after.local Compatibility
After=libvirtd.service Requires=libvirtd.service

[Service]
Type=oneshot
ExecStart=/etc/init.d/after.local
RemainAfterExit=true

[Install]
WantedBy=multi-user.target
....
Em seguida, crie o script _/etc/init.d/after.local_:

....
#! /bin/sh
#
#
# ...
echo 4 > /sys/class/infiniband/mlx5_bond_0/device/sriov_numvfs
echo 4 > /sys/class/infiniband/mlx5_1/device/sriov_numvfs
....
Após a criação, certifique-se de que o arquivo possa ser executado:

....
# cd /etc/init.d/
# chmod 750 after.local
....


== Atribuir as interfaces virtuais à máquina virtual.

Este capítulo descreve como atribuir interfaces virtuais à máquina virtual usando o _virt-manager_.

. Inicie o virt-manager. + image:kvm-hana-image2.png["largura=601, altura=385"]
. Abra a máquina virtual desejada. + image:kvm-hana-image3.png["largura=300, altura=385"]
. Pressione “Adicionar hardware”. + image:kvm-hana-image4.png["Virtmanager Adicionar Hardware, largura=601, altura=443"]
. Selecione a placa de rede virtual desejada na primeira porta física da lista de Dispositivos Host PCI e clique em Concluir.  + Neste exemplo, 0000.AB:00:2 - 0000.AB:00:4 pertencem à primeira porta física e 0000.AB:01:2 - 0000.AB:01:4 pertencem à segunda porta física. + image:kvm-hana-image5.png["largura=601, altura=442"]
. Selecione a próxima porta NIC virtual na lista de Dispositivos Host PCI, utilize uma porta virtual da segunda porta física e clique em Concluir. + image:kvm-hana-image6.png["largura=601, altura=441"]
. Em seguida, as interfaces virtuais são atribuídas à máquina virtual e esta pode ser iniciada. + image:kvm-hana-image7.png["largura=601, altura=443"]




== Configurando as interfaces de rede dentro da VM
