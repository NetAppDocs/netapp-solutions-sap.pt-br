[role="lead"]
Configure a rede Fibre Channel para SAP HANA no SUSE KVM atribuindo portas HBA físicas às VMs como dispositivos PCI.  Configure conexões FCP redundantes usando duas portas físicas conectadas a switches de malha diferentes.


NOTE: Os passos a seguir são necessários apenas se o FCP for usado como protocolo de armazenamento.  Se o NFS for utilizado, esses passos não são necessários.

.Sobre esta tarefa
Como não existe um recurso equivalente ao SR-IOV para o FCP, atribua as portas HBA físicas diretamente à máquina virtual.  Utilize duas portas físicas conectadas a estruturas diferentes para redundância.


NOTE: Uma porta física só pode ser atribuída a uma máquina virtual.

.Passos
. Iniciar o virt-manager:
+
image:kvm-hana-image2.png["largura=601, altura=385"]

. Abra a máquina virtual desejada.image:kvm-hana-image3.png["largura=601, altura=385"]
. Selecione *Adicionar hardware*.
+
image:kvm-hana-image4.png["Virtmanager Adicionar Hardware, largura=601, altura=443"]

. Selecione a porta HBA desejada na lista de Dispositivos Host PCI e clique em Concluir.
+
Neste exemplo 0000.A2:00:0.

+
image:kvm-hana-image9.png["largura=601, altura=441"]

. Selecione a porta HBA desejada na lista de dispositivos host PCI pertencentes à segunda malha e clique em Concluir. Neste exemplo 0000.A2:00:1.
+
image:kvm-hana-image10.png["largura=601, altura=441"]

. Em seguida, as portas HBA físicas são atribuídas à máquina virtual e a máquina virtual pode ser iniciada.
+
image:kvm-hana-image11.png["largura=601, altura=441"]

+
As portas físicas são repassadas para a máquina virtual, portanto, nenhuma preparação adicional é necessária dentro da máquina virtual.
