= Provisionamento e instalação do SISTEMA HANA
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


Este capítulo descreve a instalação e configuração do sistema SAP HANA específico para uma configuração VMware usando VMFS. As melhores práticas genéricas adicionais podem ser encontradas em https://docs.netapp.com/us-en/netapp-solutions-sap/bp/hana-asa-fc-introduction.html["SAP HANA em sistemas NetApp ASA com protocolo Fibre Channel"].



== Configuração de armazenamento

Para atender aos KPIs de desempenho de armazenamento definidos pela SAP para sistemas HANA de produção, LUNs e datastores dedicados devem ser configurados para os sistemas de arquivos de dados e log do sistema HANA. Os datastores não devem ser compartilhados entre vários sistemas HANA ou outras cargas de trabalho.

As ferramentas ONTAP para VMware (OTV) foram usadas para provisionar os três armazenamentos de dados para o VFS do sistema HANA.

* hana++_++dados++_++VFS
* hana++_++log++_++VFS
* hana++_++compartilhado++_++VFS



NOTE: O armazenamento de dados do sistema de arquivos compartilhado HANA também pode ser compartilhado entre vários sistemas HANA.

== Parâmetro da VM disk.EnableUUID

Este parâmetro deve ser definido adequadamente, caso contrário, a descoberta automática do banco de dados SnapCenter falhará.

. Desligar VM
. Adicione o novo parâmetro “disk.EnableUUID” e defina como “TRUE”
. Iniciar VM

....

hana-8:/ # chmod –R 777 /hana/log/SMA
hana-8:/ # chmod –R 777 /hana/data/SMA

== INSTALAÇÃO HANA

A instalação HANA agora pode ser executada.


NOTE: Com a configuração descrita, o diretório /usr/sap/VFS estará no VMDK do os. Se /usr/sap/VFS deve ser armazenado no VMDK compartilhado, o disco compartilhado HANA pode ser particionado para fornecer outro sistema de arquivos para /usr/sap/VFS.
